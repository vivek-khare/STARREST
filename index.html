What is REST ?

REST stands for Representational State Transfer. It is an architecture pattern which provides constraints for creating distributed and interoperable web services which are stateless and are uniquely identifiable by a URI. REST uses the common HTTP methods like GET, POST, PUT, DELETE and HEAD to provide the CRUD operations for the resources represented by the REST services. Go through the session recording to understand in detail about REST architectural pattern and its practical implementation using Jersey framework and Eclipse.

The Origin of REST

The term REST was coined and explained by Roy Fielding, in his PhD thesis titled 'Architectural Styles and the Design of Network-based Software Architectures'.
